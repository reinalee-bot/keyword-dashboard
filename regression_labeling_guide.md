## 기사 유형 판정 기준

### 보도자료형
기업 또는 기관이 자사 제품·서비스·행사·계약·수상·인증을 발표하는 기사.

**판정 조건 (하나 이상 해당)**
- 제목에 "출시", "체결", "수상", "론칭", "선봬", "선정", "공급계약", "수주", "파트너십 체결" 등 포함
- 기업명 + 행동어 구조의 단편 알림 (예: "○○사, △△ 개최")
- `[신간]`, `[출간]` 편집 브래킷 포함
- 신임 임원 취임 알림

**경계 케이스**
- 인증 *취득/획득* → 보도자료형. 인증 *유지/갱신* → 일반 기사
- 기업 전략 발표 기사 → 인물 발언 없으면 일반 기사; 따옴표 발언 있으면 인터뷰 우선
- 신제품 "공개"는 제품 문맥 확인 후 결정 (단순 전략 발표면 일반 기사)

---

### 기획·분석
전문가 기고, 칼럼, 사례연구, 심층 진단 기사.

**판정 조건 (하나 이상 해당)**
- `[칼럼]`, `[기고]`, `[보안 칼럼]`, `[사례연구]`, `[전문가 기고]` 등 편집 마커 포함
- 복수 출처 및 데이터 인용 구조
- "분석", "진단", "리포트" 키워드가 제목에 포함되고 단일 기업 발표 아님

**주의**: 편집 마커가 있어도 단순 수상/협약 소식이면 보도자료형 우선.

---

### 인터뷰
특정 인물의 발언이 기사의 핵심 콘텐츠인 기사.

**판정 조건 (하나 이상 해당)**
- 제목에 따옴표(", ", ') + 발언 포함
- "만났다", "말했다", "강조했다", "밝혔다" 등 직화법 표현이 제목의 주요 구조
- `[Global Security TOP 100]` 등 인터뷰 시리즈 브래킷

**경계 케이스**
- 제품 출시와 동시에 발언 따옴표가 있는 경우: 발언이 핵심이면 인터뷰, 발표 자체가 핵심이면 보도자료형

---

### 행사·현장
전시회, 컨퍼런스, 세미나 등 행사 개막·현장을 보도하는 기사.

**판정 조건 (하나 이상 해당)**
- 행사명 + "개막", "총출동", "현장", "성황" 등 포함
- "총출동"은 기업들이 행사에 대거 참가한다는 의미 (예: "삼성·LG 총출동")

**주의**: "개막"이 비유적 표현("공덕시대 개막")이면 일반 기사.

---

### 일반 기사
위 네 유형에 해당하지 않는 사실 전달형 뉴스.

예: 정책 보도, 시장 동향, 기업 전략, 사건·사고, 연구 결과 보도.

## 평가 방법

```bash
cd keyword-dashboard
python build_regression.py
```

`build_regression.py`는 현재 `news_fetcher.py`의 `classify_article_type(title, description)` 함수를 동적 로드하여 각 기사를 재분류한 뒤, `expected_type`과 비교합니다.

**주의사항**: `regression_collected.json`에는 `description` 필드가 없습니다.  
따라서 **title-only 평가**로 실행됩니다. 실제 운영 환경에서는 description 까지 사용하므로 수치가 다를 수 있습니다.  
결과는 "45건 회귀 기준 통과"로 참조하고, "운영 검증 완료"로 보지 마십시오.

### 출력 컬럼 설명

| 컬럼 | 설명 |
|------|------|
| `expected_type` | 인간 라벨 (human label) |
| `pipeline_type` | 현재 분류기 title-only 재평가 결과 |
| `stale_pipeline_type` | 수집 당시 JSON에 저장된 구(舊) 분류기 결과 (참고용) |
| `type_correct` | `pipeline_type == expected_type` 이면 Y |
| `cluster` | 동일 발표 클러스터 ID (없으면 빈 문자열) |
| `human_label_reason` | 라벨 판정 근거 (1인 판단) |

## 라벨 수정 방법

1. 판정이 달라진 기사가 생기면 `build_regression.py`의 `EXPECTED_MAP` 해당 `idx` 항목만 수정
2. `python build_regression.py` 재실행 → `regression_set.csv` 자동 갱신
3. 변경 이유를 `human_label_reason` 필드와 커밋 메시지에 기록
